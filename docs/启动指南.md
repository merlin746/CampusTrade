# 校园二手交易平台 - 启动指南

## 一、环境要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17 或以上 | 后端运行环境 |
| MySQL | 8.0 或以上 | 数据库 |
| Node.js | 18 或以上 | 前端运行环境 |
| Gradle | 项目自带 wrapper，无需手动安装 | 构建工具 |

## 二、数据库初始化

### 1. 启动 MySQL 服务

确保 MySQL 服务已启动。

### 2. 执行初始化脚本

用 MySQL 客户端（Navicat、MySQL Workbench 或命令行）执行项目中的 `init.sql`：

```bash
mysql -u root -p < src/main/resources/db/init.sql
```

或者在 MySQL 客户端中直接运行 `src/main/resources/db/init.sql` 文件。

脚本会自动：
- 创建 `school_trade` 数据库
- 创建 8 张业务表
- 插入 8 个默认商品分类
- 插入一个管理员账号占位（需要手动在应用中注册）

### 3. 修改数据库连接密码

打开 `src/main/resources/application.yml`，修改数据库用户名和密码为你的实际配置：

```yaml
spring:
  datasource:
    username: root       # 改成你的MySQL用户名
    password: root123    # 改成你的MySQL密码
```

## 三、启动后端（Spring Boot）

### 方式一：使用 Gradle Wrapper（推荐）

```bash
# 在项目根目录下执行
./gradlew bootRun        # macOS/Linux
gradlew.bat bootRun      # Windows
```

### 方式二：使用 IDE

1. 用 IntelliJ IDEA 打开项目根目录
2. 等待 Gradle 依赖下载完成
3. 运行 `SchoolTradeApplication.java` 的 main 方法

### 启动成功标志

控制台输出：`Started SchoolTradeApplication in X.XXX seconds`

后端运行在：**http://localhost:8080**

API 文档地址：**http://localhost:8080/doc.html**

## 四、启动前端（Vue3）

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖（仅第一次）
npm install

# 3. 启动开发服务器
npm run dev
```

前端运行在：**http://localhost:5173**

> 前端已配置代理，`/api` 和 `/uploads` 请求会自动转发到后端 8080 端口。

## 五、创建管理员账号

管理员账号无法通过注册页面创建（注册默认是普通用户），需要通过以下方式：

### 方式一：通过 API 文档创建（推荐）

1. 先通过前端页面注册一个普通用户账号（如 `admin`）
2. 打开 http://localhost:8080/doc.html 进入 Knife4j API 文档
3. 在 MySQL 中直接修改用户角色为管理员：

```sql
-- 将指定用户设置为管理员
UPDATE user SET role = 1 WHERE username = 'admin';
```

### 方式二：直接 SQL 插入

先启动项目一次让 BCrypt 生效，然后在代码中生成密码密文，或使用在线 BCrypt 工具生成密文后执行：

```sql
-- 密码为 admin123 的 BCrypt 密文（示例，需要用实际生成的替换）
INSERT INTO user (username, password, role, status, credit_score)
VALUES ('admin', '$2a$10$...你的BCrypt密文...', 1, 1, 1000);
```

## 六、管理员后台功能说明

管理员登录后会自动跳转到管理后台（深色侧边栏界面），包含以下功能：

### 1. 仪表盘
- 查看平台总数据：用户数、商品数、订单数
- 查看今日新增数据
- 待审核商品数提醒

### 2. 用户管理
- 查看所有注册用户
- 搜索用户（按用户名/邮箱）
- **禁用/启用用户**：禁用的用户无法登录

### 3. 商品审核 ⭐核心功能
- 查看所有"待审核"状态的商品
- 查看商品详情、图片、卖家信息
- **审核通过**：商品立即上架，状态变为"在售"
- **审核不通过**：填写拒绝原因，通知卖家

> **审核流程**：用户发布商品 → 状态为"待审核" → 管理员审核 → 通过(上架) / 不通过(退回)

### 4. 商品管理
- 查看所有商品（支持按状态筛选）
- 搜索商品

### 5. 分类管理
- 添加/编辑/删除商品分类
- 设置分类图标（emoji）和排序

## 七、核心业务流程

### 买家购买流程
```
浏览商品 → 查看详情 → 点击"立即购买" → 生成订单(待确认)
→ 卖家确认订单 → 买家确认收货 → 订单完成 → 双方互评
```

### 卖家发布流程
```
发布商品 → 待审核 → 管理员审核通过 → 上架展示
→ 买家下单 → 卖家确认 → 买家收货 → 交易完成
```

### 评价体系
- 交易完成后，买卖双方可以互相评价（1-5星）
- 评价会影响用户的**信誉分**（5星+10分，4星+5分，3星不变，2星-5分，1星-10分）
- 支持匿名评价
- 商品的平均评分由所有买家的评价计算得出

## 十、常见问题

**Q: 启动后端报 "Access denied for user"**
A: 检查 `application.yml` 中的 MySQL 用户名和密码是否正确。

**Q: 启动后端报 "Unknown database 'school_trade'"**
A: 需要先执行 `init.sql` 创建数据库。

**Q: 前端请求后端报跨域错误**
A: 后端已配置 CORS 允许所有来源，同时前端 dev server 也配置了代理。如果仍有问题，检查后端是否正常启动在 8080 端口。

**Q: 上传图片在哪里存储？**
A: 图片存储在项目根目录的 `./uploads` 文件夹下，通过 `/uploads/xxx.jpg` 访问。

**Q: JWT Token 过期时间是多久？**
A: 默认 24 小时，可在 `application.yml` 的 `jwt.expiration` 中修改。
